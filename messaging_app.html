C++ Messaging App

A Messaging App

Overview

This project was developed for my Object-Oriented Design and Analysis course at Western University as part of a five-member team. My primary contributions included implementing the UI windows, writing user stories during the planning phase, and assisting with the client-server connection setup. In addition to my own tasks, I reviewed components created by teammates—such as certain program modules and UI diagrams—and relayed mistakes I found, which helped improve overall quality. I collaborated closely with teammates to integrate features, and I was solely responsible for generating the complete Javadoc documentation. By the end of the project, we presented our work to a TA via video call, who expressed strong approval of the results.

  • UI Implementation — built and integrated UI windows with wxWidgets
  • User Story Writing — contributed to planning and requirements definition
  • Client–Server Programming — assisted in setting up client connections
  • Code & Diagram Review — identified and relayed teammate mistakes to improve quality
  • Javadoc Documentation — solely responsible for generating complete project documentation
  • Team Collaboration — worked closely with peers to integrate features and deliver results

Challenges & Learnings

A key technical challenge we faced was enabling our messaging application to connect users across different networks and devices. Initially, we had only tested the system locally on our own machines. To resolve this, our entire five‑member team worked together to research solutions and ultimately adopted ngrok to set up a server accessible externally. This collaborative approach worked reliably and allowed us to successfully showcase the project to our TA during the final video call.

  • Client–Server Networking — learned how to support connections across different networks and devices
  • Teamwide Problem‑Solving — collaborated as a five‑member group to identify and test solutions
  • Practical Use of ngrok — gained experience setting up an externally accessible server for demonstrations
  • Adaptability — adjusted project approach to ensure successful presentation under real‑world conditions
