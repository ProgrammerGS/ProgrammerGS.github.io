ABOUT

Date: ~April 2023-current

This is a game project that I have been working on and off for a few years. Its goal is to create a more immersive gaming experience through procedural animation systems, complex dialogue systems, and complex video game AI.

So far I have only made some progress in the procedural animation system, so most of this page will be showing off the work I have done there.

The procedural animation system uses inverse kinematics to allow characters (both the player and NPCs) to interact with objects in the environment in ways that are rarely seen in games. Since I intend to use the systems I create from this project for a relaxing simulation game, I believe that these animations will serve to be more fun and immersive rather than annoying and intrusive.

The steps I have planned out for this project are cumulative - many later steps require the features created in previous steps. Some of this can already be seen - where item interactions like opening umbrellas or opening doors require a basic grabbing feature first.

Tools Used: Blender, Godot

This is the most widely used application of procedural animation in video games, and can be commonly seen in games such as 3rd person shooters. I took much inspiration from such games while I was working on this step.

The videos shows basic movement (in the 4 directions) and head/camera rotation with a player-controlled character. When the player moves the camera up or down in 1st person view, the head movement is synced with the player's camera.
